### 主要技術改進

#### 1. ⭐ 注入 admin JWT 到 localStorage 跳過 RBAC

之前 CRM staff / Website 兩個 tab 因為沒登入態被 RBAC 過濾掉、整片 skip。現在 module-scoped browser context 用 `add_init_script` 在 page 第一個 script 之前注入：

```python
ctx.add_init_script(f"""
    localStorage.setItem('auth_token', {json.dumps(admin_token)});
    localStorage.setItem('auth_user', {json.dumps(cached_user)});
""")
```

`admin_token` 用 [tests/fixtures/test_data.py:make_admin_token](tests/fixtures/test_data.py) 簽 — 同個 jwt_secret，後端認得。`cached_user` 帶 `access_level: 3` + 14 個 modules，前端 `auth-state.js` 讀到後同步 hydrate state，免等 `/auth/me` async 呼叫。

#### 2. 修正 selector — 用實際 ID 不靠想像

之前用 `button[data-tab="crm"]` / `[data-crm-subtab="staff"]` 都不存在。實際是：

| 用途 | 正確 selector |
|------|--------------|
| 人力資源 tab | `button#btn_tab_crm_staff`（top-level，沒有 nested CRM 主 tab） |
| 官網 tab | `button#btn_tab_website` |
| 作品集 subview | `button[data-subview="works"]` |
| Detail panel projects tab | `#staff-detail-tabs button[data-tab="projects"]` |

#### 3. `wait_for_function` poll DOM 狀態，避免固定 sleep

`#staff-list-body` 預設有 `<div class="crm-empty">載入中...</div>` placeholder。直接 `wait_for_selector('.crm-row, .crm-empty')` 立刻 match 到「載入中」，導致拿到的是空 DOM。改用：

```python
page.wait_for_function("""() => {
    const el = document.getElementById('staff-list-body');
    if (!el) return false;
    if (el.querySelector('.crm-row')) return true;
    const empty = el.querySelector('.crm-empty');
    return empty && !empty.textContent.includes('載入中');
}""", timeout=15_000)
```

跑 staff list async fetch 完整 round-trip 才放行。

#### 4. `dispatch_event("click")` 跳過 actionability check

CRM `.crm-row` 在 RBAC apply 期間瞬間 hidden。Playwright 預設 `click()` 等 element visible/stable 30 秒會 timeout。`row.dispatch_event("click")` 直接 fire click event 觸發 inline `onclick="window._staffSelect('...')"` handler，跳過 viewport / visibility 檢查。

#### 5. ⚠ 一個沒解的 limitation：dropdown 內 closure listener

showcase-edit dropdown item 的 `mousedown` listener 是用 `addEventListener` 在每次 re-render 時 attach 的閉包。Playwright 的 `dispatch_event("mousedown")` / `mouse.down()` / `click(force=True)` 都跑不到那個 listener — 推測是 listener reference 在 `_renderStaffDropdown` 重建 DOM 時遺失。

**繞道**：把「click pick → 寫 staff_id 到 backend」的測試拆兩段：
1. UI 端只測「dropdown 重 render 出 fixture staff item」（debounce search round-trip）
2. 用 backend PUT 直接寫 staff_id，reload 後驗 render 出 input.value + 綠點

仍涵蓋全部 render 路徑，只少了實際 click → save 的端到端互動。`test_quick_add_modal_creates_and_links` 用 `dispatch_event("mousedown")` 跑得起來（可能 modal 結構 listener 不同 timing），所以 quick_add 路徑是真的端到端。
