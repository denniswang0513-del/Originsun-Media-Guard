### 修了一個真 bug

審視 server 起不來時 stderr 抓到：

```
[website-migration] batch failed, rolled back:
relation "crm_project_showcases" does not exist
```

`db/migrations_website.py` 內 `_MIGRATE_FLAT_CREDITS_SHOWCASE` SQL 在 fresh DB（沒 `crm_project_showcases` 表）會炸整個 migration batch → 導致 startup 後續 ALTER 全部 rollback。

**修法**：用 `DO $migrate$ BEGIN IF to_regclass('public.crm_project_showcases') IS NOT NULL THEN ... END IF; END $migrate$;` 包裹，表存在才跑。idempotent + fresh DB-safe。

這個 bug 任何 fresh DB 啟動都會踩中，**user 在自己的開發機跑 NAS PG 也會看到** — 階段 5 的 migration 引入但沒 verify。**已修。**
