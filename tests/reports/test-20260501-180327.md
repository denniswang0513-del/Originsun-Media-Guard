## 過程修了 5 個 fixture / 程式碼 bug

執行過程中發現原本 fixture 完全跑不起來，連帶找到 1 個 production code bug。

### 1. ⭐ Production bug：`crm_project_showcase` 表名拼錯（plural ↔ singular）

**最關鍵的修復** — 真實表叫 `crm_project_showcase`（單數），但下列檔案用 plural：

- `services/website/credit_service.py` × 5 處（SQL 查詢 / cleanup_staff_id_from_credits / find_projects_by_staff / _compute_role_usage）
- `db/migrations_website.py` × 1 處（`_MIGRATE_FLAT_CREDITS_SHOWCASE` 參數）
- `tests/fixtures/test_data.py` × 1 處（cleanup DELETE）

**影響**：
- `services/website/credit_service.py` 的 SQL 都 raise `relation does not exist` → 走 `except` 靜默 swallow → admin UI 顯示 usage_count 永遠 0、cleanup_staff_id_from_credits 永遠回 0 筆 affected
- migration `_MIGRATE_FLAT_CREDITS_SHOWCASE` 永遠 no-op（`to_regclass` 守門擋掉 → 舊 flat credits 沒升級）

**全部已修為 `crm_project_showcase`**。Production code bug，建議 publish 推 NAS。

### 2. `_self_heal_scheduled_task` 在 Session 0 殺測試 server

`main.py:120` 的 self-heal 邏輯偵測 Session 0 → schtasks 重註冊 → 4 秒後 taskkill 自己。
測試在 Session 0（pytest 不是互動 session）跑，每次 spawn uvicorn 都被 helper 殺。

**修法**：env var `ORIGINSUN_DISABLE_SELFHEAL=1` 跳過。已加在：
- `main.py:_self_heal_scheduled_task`（守門）
- `tests/conftest.py:real_server`
- `tests/test_test_skill_http.py:real_server`
- `tests/test_test_skill_ui.py:real_server`

### 3. 測試 `_get_session_factory` 跨 asyncio.run() 衝突

每個 fixture 用 `asyncio.run()` 開新 loop，但 `db.session._engine` 全局單例綁在舊 loop → asyncpg 衝突 `another operation is in progress`。

**修法**：`tests/fixtures/test_data.py:_get_session_factory` 改為每次新建獨立 engine。

### 4. `seed_test_project` 缺 `client_id`（NOT NULL violation）

`crm_projects.client_id` 是 NOT NULL（soft FK）。fixture 沒設 → INSERT 炸。

**修法**：seed 時帶 `client_id="__test_dummy"`。

### 5. UI test `asyncio.run()` 在 pytest-asyncio auto loop 裡炸

`pytest.ini` 設 `asyncio_mode=auto` → 所有 test 包進 event loop → `asyncio.run()` raise `cannot be called from a running event loop`。

**修法**：`tests/test_test_skill_ui.py` 加 `_run(coro)` 透過 ThreadPoolExecutor 開獨立 thread + 獨立 loop。
