# Originsun Media Guard Pro SaaS - 維護與發布指南

這份文件是專門寫給你（系統管理員 / 開發者）的。
當你未來修改了任何程式碼（例如增加了新功能、換了新網頁介面、修了 Bug），你 **不需要自己手動壓縮檔案給同事**，只要照著這份指南操作，系統就會自動幫你發布給所有人。

---

## 🚀 1. 如何發布新版本給同事？

當你修改完程式碼，並且在伺服器上測試沒有問題後，請開啟終端機 (Terminal /命令提示字元)，並確認路徑在 `D:\Antigravity\OriginsunTranscode` 底下，然後輸入：

```bash
python publish_update.py
```

### 腳本執行流程：

1. **版本號確認**：腳本會顯示目前的版本號（例如 `1.0.1`），並問你新版號是什麼。你可以輸入 `1.0.2` 或直接按 `Enter` 沿用舊版號。
2. **更新日誌**：腳本會問你這次更新了什麼（例如輸入 `修訂了 UI 排版`）。
3. **自動打包**：腳本會自動把所有必須的檔案（包含你剛寫的程式碼、網頁檔以及 FFmpeg 轉檔引擎）全部壓縮成一個大包裝 `Originsun_Agent.zip`。
4. **自動推送到 NAS**：腳本會詢問你「是否要立即同步檔案至 NAS? (y/n)」，請直接按 `Enter` 或是輸入 `y`。腳本會自動把最新的程式碼複製到你的共用資料夾（`\\192.168.1.132\Container\AI_Workspace\agents\Originsun Media Guard Pro`）。

**🎉 只要看到「發布成功！NAS 端已擁有最新版」，你的工作就結束了！**

## 👩‍💻 2. 同事那邊會發生什麼事？

你發布新版本後，所有的派發與更新都是 **全自動** 的：

### A. 已經安裝過舊版代理的同事（熱更新 OTA）
當你完成上述的 `publish_update.py` 發布動作後：
1. 長期開著網頁的同事，在下次點擊或刷新網頁時，網頁會自動到 NAS 檢查 `version.json`。
2. 網頁發現 NAS 的版本比同事電腦裡的版本新。
3. 網頁右上角原本綠色的「本機節點在線」按鈕，會變成閃爍的紅色火箭：**「🚀 發現新版本 (v1.0.2)」**。
4. 同事點擊該按鈕後，背景會觸發 `update_agent.bat`。它會默默去 NAS 複製你剛寫好的新程式碼，然後自動重新啟動代理，五秒內完成無痛升級。

### B. 從來沒安裝過的新同事（零安裝體驗）
1. 新同事打開你的網頁（`http://192.168.1.11:8000`）。
2. 首頁會跳出紅色警告「🔴 未偵測到本機代理」。
3. 同事點擊下載精靈。
4. 下載到一個很小的 `Install_Originsun_Agent.bat`，點兩下執行。
5. 該精靈會自動去你的伺服器抓取你剛剛發布的最新版 `Originsun_Agent.zip` (內涵 150MB 的完整運算環境與 FFmpeg)，然後自動解壓縮並丟在桌面上，隨後立即開工。

## ⚠️ 3. 常見維護問題與解法

- **我要加新的 Python 依賴套件怎麼辦？**
  如果你 `pip install` 了新的套件，不用擔心，打包腳本會自動把整個 `.venv`（虛擬環境）包裝進 `Originsun_Agent.zip` 裡面，新同事下載會直接拿到新套件；舊同事因為是走 OTA 差量更新，目前腳本沒有每次更新 `.venv`，如果你的更新非常重大，建議請同事「**重新下載精靈重新安裝**」一次。
  
- **我不小心把版本號寫錯了怎麼辦？**
  你可以在專案目錄下找到 `version.json` 這個檔案，用純文字編輯器直接手動改回你要的數字，然後再跑一次 `publish_update.py` 發布覆蓋 NAS 的檔案即可。

- **檔案同步到一半斷線或報錯怎麼辦？**
  請檢查你的伺服器（正在跑腳本的這台電腦），是否能正常連線到網路磁碟 `\\192.168.1.132`。如果網路沒問題，直接再跑一次 `python publish_update.py` 就好。
