# 員工管理（N-hr）與財務報表（N-fin）規劃（2026-07-08）

> 定位：Work OS Phase N 的兩條補完支線。**邊界先講清楚**：
> - N-hr 不做 HRIS——勞健保申報、薪資計算外包給會計師/薪資系統；本系統只出
>   「工時 + 獎金結算單」給會計，管的是**檔案、假勤、派工適配、組織記憶**。
> - N-fin 不做記帳軟體——不做借貸分錄/總帳；本系統做**管理報表**（損益/毛利/
>   帳齡/客戶貢獻），數字給老闆做決策，正式帳還是會計師的。
> 實作慣例照舊：CRM checklist、RBAC 3 處、錢流純函式進 `core/crm_logic.py` + 測試、canary。

---

## N-hr：員工管理（長在 crm_staff + N0 上）

### H1. 員工檔案完整化（無依賴，可先做）

- `crm_staff` 補欄位：employment_type（正職/兼職/約聘/freelance）、hire_date、
  leave_date、emergency_contact、id_number（加密存）、note
- **`staff_rate_history`**（關鍵設計）：staff_id, day_rate, effective_from —— 日費率
  調整不可改寫歷史；工時成本回寫（N2）與估價複盤（B2）都要按「當時費率」算，
  否則歷史毛利全部失真
- `staff_skills`：技能/證照 tags（空拍證照+效期、攝影、剪輯、調光、收音…），
  證照效期到期前提醒（notifier 現成）
- `staff_documents`：合約/保密協議上傳（uploads/staff/{id}/，僅 admin + 本人可見），
  合約到期提醒

### H2. 請假與假勤（依賴 N0 極簡審核機制）

小公司極簡版，**不做打卡鐘**（工作紀錄由 N2 工時覆蓋，不重複）：

- `leave_requests`：staff_id, type（特休/病假/事假…）, start/end, days, reason,
  status —— 審核走 N0「狀態+核可人+時間戳」共用模組
- 假別額度：staff 檔存年度特休天數，餘額即時算（申請合計即可，不用另存）
- 排班日曆（N2）自動顯示請假 → 派工避開；「我的假勤」進 N4 我的頁

### H3. 派工適配（掛在 N2 排班內實作，此處定資料）

- 排班選人時按 `staff_skills` 篩選 + **檔期衝突檢查**（該日已派其他案 → 標紅）
- 外部協力（freelance）同一張表管理（employment_type 區分），報價/請款聯動已有

### H4. 協力評價（組織記憶，同場景庫「踩雷紀錄」哲學）

- `staff_reviews`：staff_id, project_id, rating(1-5), note, created_by —— 結案時
  PM 對外包/協力快速評分（N1 結案 hook 順帶提示）
- 人員詳情顯示歷史評價聚合 →「誰靠譜」不再只存在老闆腦中；敏感欄位僅 Lv3 可見

### H5. 到職/離職 checklist

- 模板化 checklist（到職：開帳號/給模組權限/器材領用(B4 聯動)/加入群組；
  離職：收回全部反向）→ 一鍵生成公布欄任務組
- 離職自動盤點：該員名下未歸還器材（B4）、未結預支款（帳務現成）、進行中派工

**不做**：薪資計算、勞健保、打卡、KPI 考核系統（太重，小公司用不到）。

## N-fin：財務報表（長在既有帳務五視圖上）

### F1. 科目整理 + 月結機制（地基，建議最先做）

- **極簡科目表**：`finance_categories`（收入/外包/人事/器材/行政/稅費…兩層就夠），
  cash_entries 的 category 對映到科目——沒有科目一致性，所有報表都是流沙
- **月結鎖帳**：`finance_month_close`（month, closed_by, closed_at, snapshot JSON）——
  鎖定月份的收支不可改（要改須 reopen + 記 audit log）；報表數字可重現、
  可回溯，這是「報表可信」的地基
- 月結時自動跑檢核：未分類項目、未關聯專案的大額支出、未收款逾期清單

### F2. 月度損益表（P&L）

- 收入（發票/實收可切換基準）− 外包成本 − 雜支 − 固定成本（settings 月固定成本，
  B3 共用）→ 月毛利/淨利 + 12 個月趨勢圖
- 純聚合查詢 + `core/crm_logic.py` 純函式（可測試）；儀表板卡 + 明細下鑽

### F3. 專案毛利報表（與 B2 估價複盤共用底層）

- 每專案：合約額 − 外包 − 雜支 →（N2 後 + 工時成本；B4 後 + 器材折舊）= 毛利/毛利率
- 初版**現在就能算**（合約/外包/雜支都在），N2/B4 上線後自動變準
- 排行視圖 + 類型別/客戶別彙總；毛利率低於門檻標紅

### F4. 客戶貢獻與集中度

- 年度營收 by 客戶 + 前三大客戶占比——**占比 > 50% 亮警示**（接案公司的
  隱形風險：大客戶抽單 = 現金流斷崖）；搭配客戶狀態自動化既有邏輯

### F5. 應收帳齡（Aging）

- 30/60/90 天分桶 + 逾期清單（B3 做「提醒」，F5 做「歷史視圖」，共用資料）

### F6. 報表輸出與自動化

- 每張報表：儀表板卡 + CSV 匯出；月報 PDF 復用報價 PDF 管線（Jinja2 + playwright，
  ROADMAP 既有項一石二鳥）
- **月結自動化**（STRATEGY 自動化梯隊 #3 在此落地）：每月 1 號 runner 產上月
  P&L + 帳齡 + 專案毛利摘要推 Google Chat（webhook 前置又+1）

### F7. 稅務準備包（小公司剛需）

- 年度/季度打包：發票彙總表 + 憑證清單 + 分類支出統計 → CSV/PDF 一鍵匯出給會計師
- 省的是每季翻單據的兩天人工

## 建議順序與依賴總表

| 段 | 內容 | 依賴 | 粗估 | 何時做 |
|----|------|------|------|--------|
| F1 | 科目 + 月結鎖帳 | 無 | 1-2 session | **N-fin 地基，最先** |
| F2+F5 | P&L + 應收帳齡 | F1 | 1-2 session | F1 後接續 |
| F3+F4 | 專案毛利初版 + 客戶集中度 | F1 | 1 session | 接續（N2/B4 後自動變準） |
| F6+F7 | 月報自動化 + 稅務包 | F1-F5、webhook | 1-2 session | 收尾 |
| H1 | 員工檔案 + 費率歷史 + 技能 | 無 | 1-2 session | **N2 排班前必須完成**（rate history 是 N2 成本回寫的前置） |
| H4 | 協力評價 | N1 結案 hook | 0.5 session | 搭 N1 |
| H2 | 請假審核 | N0 審核機制 | 1 session | N0 後 |
| H3 | 派工適配 | H1 + N2 | — | 併入 N2 排班實作 |
| H5 | 到離職 checklist | B4 器材(盤點項) | 0.5 session | 穿插 |

**關鍵路徑提醒**：H1 的 `staff_rate_history` 是 N2 工時成本回寫與 B2 估價複盤的
共同前置——**沒有費率歷史，之後所有成本數字都會被「現在的費率」污染**，這一張表
建議在 N2 動工前先落地。F1 月結同理是 N-fin 全部報表的地基。
