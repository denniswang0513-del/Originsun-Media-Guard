## 0. 系統現在是什麼（一段話）

一套自建的影像公司作業系統：**素材線**（備份/轉檔/驗證/串接/報表/語音辨識/TTS，
主控+機隊 OTA）、**業務線**（CRM 六模組：客戶/專案/報價/人力/帳務/預支結算）、
**行銷線**（Astro 官網 + AI SEO / 英文翻譯 / 社群文稿三個每日 runner + 審核工作台）、
**協作線**（公布欄 = 人與 Claude 共用交辦收件匣）。行銷線已接近「零邊際成本的
自動行銷部門」——這是目前最大的差異化資產。

3. **職務視角首頁**：RBAC v2 已按帳號綁模組，補一層「角色聚合頁」——製片看案況
   與通告、剪輯看佇列與素材、會計看帳務、老闆看儀表板。每個職務進來只看到自己的頁。

5. **N-biz 商業功能線**（見 [`BIZ_PLAN.md`](BIZ_PLAN.md)，穿插做）：B3 現金流預測+

   recurring revenue）、B4 器材折舊、B5 素材庫、B6 客戶關懷 runner；B2 估價複盤鎖 N2 後。
6. **平台強韌**：restore 演練一次（備份沒演練過 = 沒有備份）；評估生產 master
   搬離 dev 共存機；**Win10 已過 EOL，Win11/新機計畫要排**。

## 3. 長程（2027+）：商業選項（誠實評估）

- **選項 A：內部護城河（預設路線，推薦）** — 不賣軟體，賣「比同業快且省的產能」。
  自動行銷部門 + 財務透明 + 交付流水線 = 接案毛利優勢。量化三個指標：
  詢問→成案轉換率（儀表板已有）、結案→上架時滯（目標 <48h）、每案毛利率（CRM 已可算）。
- **選項 B：切片產品化（次推薦）** — 若要外部營收，挑**一個窄切片**賣給同業，
  最有賣相的是「影像公司作品行銷自動化」（官網模板 + 結案上架 + AI SEO/翻譯/社群 runner）。
  閘門條件：**先有 2-3 家同業願意付費試用再投開發**，不要先寫 code。
- **選項 C：全平台 SaaS（現階段不推薦）** — 多租戶隔離、資安、客服支援是重寫級投入，
  會吃掉本業產能。除非選項 B 驗證出真實付費需求才升級考慮。

## 4. 自動化藍圖

**已在跑**：AI SEO / 英文翻譯 / 社群文稿三個每日 runner、空拍排程監控、
書籤+cron 排程、公布欄 Claude 交辦、機隊 OTA + busy check、告警系統（缺 webhook）。

**下一梯隊（按 ROI 排序）**：

1. 結案 hook 自動開官網上架任務（做中，N-now）。
2. **素材進線自動化**：備份完成 → 自動轉檔 + Whisper 逐字稿 → 標記候選 highlight
   ——後期剪輯直接省掉找素材的時間，是素材線最大提效點。
3. **月結自動化**：收支/應付/應收月報自動生成推 Google Chat（CRM 資料都在，缺聚合排程）。
4. **客戶交付通知**：報表/成品連結自動寄送（SMTP 基礎設施已有，聯絡表單在用）。
5. **週報 digest**：每週一推「系統健康 + 業務數字」到 Chat，讓自動化的存在感可見。

**自動化守則（社群 runner 已示範，之後一體適用）**：每個自動化必有
kill switch、量上限、審核佇列或事後可稽核 log；外發內容（社群/email）預設先審後發，
影子運行校準後才全自動。
